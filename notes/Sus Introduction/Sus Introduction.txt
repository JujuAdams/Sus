Sus 0.0.0, 2026-04-16

Sus is a user management library built to be easily deployed for strictly single-player games
cross-platform. This library handles many platform quirks and porting problems for you. In my
career as a porting engineer I have solved the same problems again and again; Sus is a simple tool
that I find useful whilst doing my job.

If your game has any multiplayer modes - including local multiplayer - then this library is not
suitable. This library is also not guaranteed to be fully compliant and is not a magic spell that
will get your game through compliance checks with zero effort on your behalf.

Despite Sus doing a good job of providing a consistent API, each platform inevitably works slightly
differently. Please follow the instructions below per platform to ensure that Sus has been imported
correctly into your game. You should also read the `Sus Platform Notes` for further information
pertinent to use of the library.

Sus requires the use of Input (https://codeberg.org/offalynne/Input) for all platforms. Input is a
well-tested cross-platform input system that Sus relies upon to accurately track gamepads.

Steam:
  1. Import YoYoGames' official Steamworks extension:
       https://github.com/YoYoGames/GMEXT-Steamworks
     You only need the `Steamworks` extension asset and the `Steamworks_Definitions` scripts (you
     don't need the example assets).
  2. Set the `Application ID` for your game in the Steamworks extension options. You can find the
     application ID for your game in the Steam backend.
  3. Remove existing calls to `steam_update()` and `steam_is_screenshot_requested()` from your
     project. Sus handles these function calls for you.
  4. Import the Sus .yymps
  5. Test!

  Protip:
    If you want to test prior to paying to release on Steam, use `378090` which is a free game
    called "Rebel Wings". You will need to obtain a copy of this game on Steam too:
      https://store.steampowered.com/app/378090/Rebel_Wings/

Switch:
  1. Make sure your game boots on devkit hardware before proceeding. You will need to refer to
     YoYoGames and Nintendo documentation for this.
  2. Remove existing calls for the "controller support applet" (CSA) from your project.
  3. Set the following properties in your .nmeta:
       - "Console account selection at startup" to `Required`
       - "Console account selection at startup option" to `Not specified`
       - "Restriction on switching user accounts" to `Enable`
     Please refer to Nintendo's documentation for more information
  4. Remove existing calls to `switch_controller_support_show()` (you will instead need to call
     the Sus function `SusSwitchShowCSA()` to show the CSA manually).
  5. Import the Sus .yymps
  6. Open the `__SusConfigSwitch` script and customise it to your liking. Please refer to 
     YoYoGames' documentation for more information.
  7. Test!

PlayStation 5:
  1. Make sure your game boots on devkit hardware before proceeding. You will need to refer to
     YoYoGames and Sony documentation for this.
  2. Remove existing calls to `psn_tick()` from your project. Sus will call this function for you.
  3. Enable `InitialUserAlwaysLoggedIn` in your params.json. Please refer to Sony's documentation
     for further information on how to do this.
  4. Import the Sus .yymps
  5. Ensure the Input config macro `INPUT_PS5_SINGLE_USER` is set to `true`.
  6. Test!

Xbox Series X/S:
  1. Make sure your game boots on devkit hardware before proceeding. You will need to refer to
     YoYoGames and Microsoft documentation for this.
  2. Remove existing calls to `xboxone_find_controller_for_user()` from your project. Sus will call
     this function for you.
  3. Enable `Simplified User Model` in your Xbox One & Series X/S game options in the GameMaker IDE
  4. Import the Sus .yymps
  5. Ensure the Input config macro `INPUT_XBOX_SIMPLIFIED_USER_MODEL` is set to `true`
  6. Test!

Windows GDK (Xbox on Windows):
  1. Import YoYoGames' official Windows GDK extension:
       https://github.com/YoYoGames/GMEXT-GDK
     You only need the `GDKExtension` extension asset.
  2. Enable `Auto Initialise` in the GDK extension's options. This removes the need to call for you
     to call `gdk_init()` yourself.
  3. Make sure your game boots using the GDK extension before proceeding. You will need to refer to
     YoYoGames and Microsoft documentation for this.
  4. Remove existing calls to `gdk_update()` and `gdk_quit()` from your project. Sus will call
     these functions for you.
  5. Import the Sus .yymps
  6. Test!

Other platforms:
  Sus does not have support for these platforms yet.
